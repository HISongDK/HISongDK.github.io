<深夜凌晨>
写代码又碰见个问题，可能不是个 bug，但是让我懵逼的调了估计也快一个小时才发现。react 组件接收的 props，有延迟，我是直接把一个值拿来 useState 初始化状态了。状态一直是空数组，打印那个 props 明明有值。就莫名其妙不知道什么问题，快一个小时才想到可能有延迟，查了下果然啊。

[解决思路](https://juejin.cn/post/6844904163973791758)还是看别人处理 vue 相同的问题才知道。vue 是用 watch 侦听属性里面监听状态重新给状态赋值。react 里面就是用 useMemo 监听依赖项处理赋值。问题是我之前一直以为 useMemo 是一定要返回一个新的状态值的，我给理解成了它和 vue 的 compute 计算属性是一个东西了。完全没想到这玩意不返回新状态也行，这玩意就同时相当与 vue 的计算和侦听属性了

还有昨天调了一点 niz 键盘符合 mac 和我自己的键位映射也是记一下比较好：

因为习惯 vim 的操作方式，一版会做一点方便的键位映射。windows 上是直接修改系统注册表，把 `Esc` 和 `CapsLock` 互换。相对来说 mac 上面我直接通过设置把 `中/英` 改成了 `Esc` 。

这样来说其实就可以正常操作了，vim 插件 `insert` 模式返回 `normal` 模式时切换英文输入法暂且不提。

不过我还有两把外接键盘，而那把 niz 的是我一般工作时候用的。它的 mac 键位又有点问题，`CapsLock` 的键位没有变，所以设置上这个键并没有指定成 `Esc` ，还是没法用。

不过好在 niz 有两种模式： 编程模式、常规模式。

可以通过 fn + Pause 键切换。编程模式下的键位可以通过软件指定一些自定义的调整，我就把键盘的编程模式下的 `Esc` 和 `CapsLock` 又互换了。

因为我的 windows 设备是直接修改的内部注册表键位，也没有什么其他需要调整的，所以 niz 就用正常模式就行。

mac 上面，就需要先切换成 mac 模式，再打开编程模式。

其实还有点 option 和 command 键位还是和正常配列不一致的问题。不过话说回来还是先不管了，这键盘毕竟一时半会也用不上。要用上的话差不多还得配个电脑支架，要不然放在上面会比较高，没法用，要是配了支架那也得再弄个蓝牙鼠标了，不然总伸手够支架上的触摸板也是够麻烦的。
